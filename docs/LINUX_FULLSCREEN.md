# Linux 全屏策略

Linux 桌面环境差异很大，FocuSeed 必须把“全屏请求”和“强制占用屏幕”区分开。

## 普通桌面模式

普通桌面模式只做最佳努力：

- 请求窗口全屏。
- 请求无窗口装饰。
- 在支持的窗口管理器上请求置顶。
- 检测当前桌面环境并显示能力提示。

## GNOME / Wayland 限制

Ubuntu GNOME 等 Wayland 桌面通常不允许普通应用：

- 强制覆盖系统顶栏、概览、快捷键或其他应用。
- 阻止用户切换窗口。
- 永久保持高于所有系统界面。

因此 FocuSeed 在 GNOME/Wayland 下应显示“最佳努力全屏”提示，并建议严格用户使用 Kiosk 会话。

## 检测信息

桌面端应采集并展示这些诊断信息：

- `XDG_CURRENT_DESKTOP`
- `XDG_SESSION_TYPE`
- `DESKTOP_SESSION`
- 是否成功进入全屏。
- 是否支持置顶请求。

## 严格专注建议

严格 Linux 模式应通过系统级部署实现：

- 创建专用用户。
- 配置自动登录。
- 启动单应用桌面会话。
- 禁用不必要快捷键。
- 限制终端、应用启动器和系统设置入口。
