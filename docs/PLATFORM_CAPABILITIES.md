# 平台能力边界

FocuSeed 的“强制专注”依赖操作系统公开能力。仓库实现时必须把能力分成普通模式、授权增强模式和系统托管模式，不能把系统不允许的能力描述成普通应用一定可以完成。

## Android 10+

### 可以直接实现

- 使用沉浸式全屏隐藏状态栏和导航栏。
- 使用 `ACTION_DIAL` 打开系统拨号器。
- 通过 `PackageManager` 查询支持拨号 Intent 的应用。
- 通过生命周期回调在用户返回应用时恢复番茄钟界面。
- 使用真实时间锚点保持计时持续推进。

### 需要用户授权

- 屏蔽通知需要用户授予勿扰模式访问权限。
- 读取通话状态需要运行时权限，并且不同 Android 版本会有差异。
- 让电话在勿扰模式下例外，需要配置 Notification Policy 中的 calls 例外。

### 需要系统托管或设备策略

- 真正限制用户只能留在 FocuSeed 和电话应用内，需要 Lock Task、应用固定、设备所有者或企业/Kiosk 策略。
- 在托管设备上，应把检测到的默认拨号器和必要系统电话组件加入 allowlist。

### 不应承诺

- 普通应用不能静默接听电话。
- 普通应用不能无条件禁止用户按 Home、最近任务或切换到其他应用。
- 普通应用不能保证所有厂商 ROM 都按同一种方式显示通话界面。

## Windows

### 可以直接实现

- 创建无边框全屏窗口。
- 保持窗口置顶。
- 使用真实时间锚点保证失焦期间计时继续推进。

### 需要系统配合

- 阻止 Alt+Tab、Win 键、任务管理器或其他应用切换，需要 Assigned Access、Kiosk 模式或管理员策略。
- 通知屏蔽通常需要用户或系统策略配置 Focus Assist/Do Not Disturb。

## Linux

### 可以直接实现

- 请求窗口全屏。
- 在支持的窗口管理器上请求置顶或无装饰窗口。
- 检测 `XDG_CURRENT_DESKTOP`、`XDG_SESSION_TYPE` 等环境信息并给出提示。

### 关键限制

- Wayland 设计上禁止普通应用强制控制全局窗口层级。
- GNOME Shell 可以拒绝或限制置顶、独占全屏、覆盖面板等行为。
- Ubuntu GNOME 等桌面环境下，普通应用可能无法真正充满所有屏幕区域或阻止用户切换。

### 推荐严格方案

- 为 Linux 严格专注模式提供单独 Kiosk 会话文档。
- 在受控设备上使用专用用户、自动登录、单应用会话和系统级快捷键限制。
- 对普通桌面用户显示“最佳努力全屏”提示，而不是承诺强制锁定。
